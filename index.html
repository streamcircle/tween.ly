Where Broadcast Design Comes to Life

Animation tool for broadcast graphics

With over 30 years in broadcast technology, we’ve seen the same challenge across TV stations of all sizes: the need to create graphics with real-time data, tickers, and API control — and export them as production-ready HTML. That’s why we built Tweenly — a web-based application tailored to the specifics of broadcast graphics. It streamlines design, data connection, and playout integration in one powerful tool.

Solution

Right Broadcast Tools

Take full control of your broadcast graphics design from scene creation to playout integration. Tweenly allows you to spend more time creating dynamic tickers, lower thirds and titles, and less time dealing with technical challenges. The appliactaion is fully cloud-based — access your graphics, share, and export from anywhere.

Editor

The Editor is where you build individual scenes. Each scene consists of objects — such as shapes, texts, and images — that can be nested, grouped, and positioned in layers. Styling is handled through CSS. The timeline allows you to animate objects and control their timing. Triggers and labels define programmatic behavior during playback.

Gallery

The Gallery is a personal cloud-based storage for your graphics. It contains all created or imported scenes and allows browsing, editing, versioning, and exporting. Every project remains accessible and editable at any time.

Workspaces

Workspaces are shared environments where multiple users can collaborate on graphics. Each workspace contains a set of scenes, assets, and configuration data. Access rights and roles determine who can view, edit, or export content. All changes are synced in real time.

Graphics export options

Graphic files can be exported as plain HTML with files linked to our cloud, or as a ZIP package that includes all assets needed for offline use. The package format is especially recommended when working with images, custom fonts, or image sequences.

Tickers

Display scrolling or crawling text within a scene. Their direction, speed, and content can be configured for different use cases, such as sports results or news headlines.

Learn more →

Image Sequence

An image sequence is a series of numbered images used for frame-by-frame animation. Tweenly supports importing and timing these sequences.

Learn more →

Code Triggers

Triggers are code blocks executed during timeline playback. Timeline labels mark specific timeline points and are used to pause, jump, or sync animations programmatically.

Learn more →

Data Source & API Control

Text, image, and ticker fields can be updated from external sources. This includes connecting to structured data feeds (JSON, XML, TXT) or receiving values via API commands from a playout system. No scripting is required. Input values are applied in real time during playback.


Learn more →

Build-in Test App

The Test App simulates API inputs to preview graphics under different conditions. It allows manual testing of text, images, and styles before playout.
